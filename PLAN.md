# CalLens 统计页改进任务

## 项目背景

CalLens 是一个基于 Lyubishchev 时间记录方法论的日历统计应用。当前统计页（统一路由下的 `周/月/季/年/全部` 视图）已有以下模块：总览卡片、时间分配（分类分布 + 每日分布）、柳比歇夫分析（Type I/II 分离 + 分类累计）、节奏与日程（24小时能量分布 + 每周节奏 + 周度热力图）、时间预算（预算 vs 实际 + 超预算/未超预算）、趋势与对比（30天滚动 + 周环比 sparkline）、本周回顾、值得注意的时刻。

设计语言保持现状：克制、米色基调、Lyubishchev 风格的"安静"信息密度。**不要引入新的视觉风格，所有改动遵循现有 design tokens。**

## 核心原则

1. **数据诚实优先于视觉完整**：宁可隐藏一个模块，不要展示误导性数字。
2. **从描述走向反思**：UI 文案不只陈述事实，要指出张力、偏差、对比。
3. **冷启动期友好**：新用户前两周看到的不是"假数据"，而是"积累中"的诚实状态。

---

## 任务清单（按优先级）

### P0 - 数据成熟度系统（最高优先级，先做这个）

引入"数据成熟度"概念，根据有效记录天数控制各模块行为。

**实现要求：**

1. 新增工具函数 `getDataMaturity(records)` 返回：
   - `daysRecorded`: 有记录的不同日期数
   - `consecutiveDays`: 当前连续记录天数
   - `maturityLevel`: `'cold' | 'warming' | 'mature'`
     - cold: < 3 天
     - warming: 3-13 天
     - mature: ≥ 14 天

2. 各模块按成熟度降级：

| 模块                     | cold                           | warming                              | mature   |
| ------------------------ | ------------------------------ | ------------------------------------ | -------- |
| 总览卡片"↑X.Xh"环比标签  | 隐藏                           | 显示但加灰色"参考"标记               | 正常     |
| 周环比 sparkline 区块    | 整块隐藏，替换为"积累中"占位卡 | 显示但 +100% 这类极值标灰            | 正常     |
| 30天滚动趋势             | 整块隐藏                       | 显示但加"数据较少，趋势仅供参考"水印 | 正常     |
| 每周节奏（每日主导活动） | 隐藏                           | 只显示真实有记录的日期               | 正常     |
| 柳比歇夫分析             | 显示，但年度推演隐藏           | 显示完整                             | 显示完整 |

3. 占位卡组件 `<MaturityPlaceholder />`：
   - 显示进度环（X/14 天）
   - 文案："继续记录 N 天，解锁趋势分析"
   - 视觉风格沿用现有卡片，但内容区域降低对比度

### P1 - 预算与回顾打通（让"本周回顾"真正反思）

**当前问题**：本周回顾文案是描述性的（"庶务时间占据了最多时间，达到20.0h"）。

**改进**：让回顾引用预算系统的数据，输出比例性、张力性的句子。

**实现要求：**

1. 在生成回顾文案的逻辑里读取用户预算配置（`weeklyBudget`），生成至少包含以下三类句式之一：
   - **比例对比**："主要矛盾 8h vs 庶务时间 20h，比例 1:2.5——你的目标是 1:4 倾向核心创造。"
   - **预算偏离**："庶务时间超预算 +15h（实际 20h / 预算 5h），是本周最大的预算溢出。"
   - **趋势张力**："核心工作 8h 比上周增加，但庶务时间增长更快（+20h vs +8h），核心占比反而下降。"

2. 文案生成函数 `generateWeeklyReflection({ stats, budget, previousStats, maturity })`：
   - cold 期：只输出第一句"本周净有效时间为 X.Xh"，不做对比
   - warming 期及以上：输出 2-3 句反思
   - 保留尾部斜体引语"记录让不可见变得可见"，但允许从一个池子里随机选

3. 文案规则：
   - 任何超预算 ≥50% 的分类，必须在回顾里被提及
   - 任何"零记录"的预算分类（个人提升 0.0h / 5h），也必须被提及
   - 用对比句式（"vs"、"是...的 N 倍"、"高出/低于"），避免单纯陈述数字

### P2 - 柳比歇夫年度推演

**当前问题**：柳比歇夫方法的真正震撼在于长期累积，但当前 UI 只显示本周/本月。

**实现要求：**

1. 在"柳比歇夫分析"模块的 Type I/II 分离卡下方增加"年度推演"小卡：
   - 公式：`projected_yearly = current_period_avg × 52 (周) 或 × 12 (月)`
   - 文案："按当前节奏，全年 Type I 将累积 **416h**"
   - 加一行 benchmark 注释（可硬编码）："Lyubishchev 1966 年 Type I：约 2200h"
   - 仅在 maturity ≥ warming 显示

2. "核心工作 8.0h" 这张总览卡，在卡片底部小字加上下文：
   - 格式：`相当于全年 416h · 柳比歇夫年均的 19%`
   - 仅在 maturity ≥ warming 显示

### P3 - 空白时间可见化

**当前问题**：周度热力图只显示有记录的格子，但"没记录的时间去哪了"才是更值得反思的。

**实现要求：**

1. 总览区新增一张"时间去向"卡（与现有四张并列或单独一行）：
   - 三段式横条：`已记录 (34h) | 睡眠 (56h) | 未记录 (78h)`
   - 三种颜色用现有 token，"未记录"用最淡的灰
   - 副标题："本周 168 小时如何分配"
   - 点击"未记录"段，可展开看哪些时段缺失（按天×小时矩阵）

2. 周度热力图增加显示模式切换：`记录密度 | 空白分布`
   - 默认仍是当前的记录密度
   - 切换后显示空白格（无记录的时段），颜色反转

### P4 - 记录质量元指标

**当前问题**：当前所有指标都是关于"记录的内容"，没有关于"记录这个行为本身"的指标。

**实现要求：**

1. 新增模块"记录质量"，放在"值得注意的时刻"上方或并列：
   - **记录条数**：本周共 N 条记录
   - **平均颗粒度**：每条记录平均 X.X h（颗粒度越细越好，给出 benchmark）
   - **记录时延**：实时记录 vs 事后补录的比例（需要数据模型支持创建时间 vs 事件时间）
   - **覆盖率**：有记录的时段 / 清醒时段

2. 这是元数据模块，视觉上要弱化（小字、低对比度），不喧宾夺主。

### P5 - 预估 vs 实际（校准模块）

**当前问题**：整个统计页是回看型的。Lyubishchev 方法的精髓是**校准对时间的感知偏差**。

**实现要求：**

1. 新增轻量的"预估 vs 实际"模块（位置：周回顾之后，作为下一周的入口）：
   - 周一首次访问统计页时，弹出（或顶部 banner）让用户预估本周各分类时间
   - 周日（或本周结束后）显示对比：每个分类的"预估 / 实际 / 偏差"
   - 偏差超过 ±30% 的分类高亮

2. 数据模型增加 `weekly_estimates` 表：`{ week_id, category_id, estimated_hours, created_at }`

3. 偏差类型分类：
   - 系统性高估（持续多周高估同类）→ 文案："你倾向于高估 [核心工作] 时间，已连续 3 周"
   - 系统性低估
   - 偶发偏差（不报警）

### P6 - 视觉与交互细节

打包修复以下小问题：

1. **顶部"对比 / 环比"下拉**：对比锚点不明显。改为显式标签，例如 `对比：环比上周（10.20-10.26）` 而不是单纯"环比"。

2. **超预算的视觉强度不够**：当前"庶务时间 20h / 5h，+15h" 仍然是普通进度条。改为：
   - 进度条超过 100% 时，超出部分用斜纹（CSS `repeating-linear-gradient`）
   - 加一个小的警示图标（不要 emoji，用 lucide 的 `AlertTriangle`）
   - 数字 `+15h` 用 token 中的 warning/error 色

3. **"最长连续记录段：10.0小时 — 睡觉"**：把睡眠分类从"连续记录段"统计里排除（或加配置项）。增加配置 `excludeFromStreaks: ['睡眠']`，默认排除。

4. **左侧导航栏**：当前全是无文字图标。增加 hover 展开（200ms delay），展开后显示中文标签。或者首次进入产品时默认展开，用户手动收起后记忆状态。

5. **环比 +100% 标记**：当上一期为 0 时，不显示百分比，改为显示绝对值变化 `+8.0h（首次记录）`。

---

## 实现顺序建议

1. 先做 **P0**（数据成熟度）—— 这是地基，其他改动都依赖它
2. 再做 **P6** 中的小修复（5 个都是低风险高回报）
3. 然后 **P1**（回顾打通）—— 用户感知最强
4. 之后按 P2 → P3 → P4 → P5 推进
5. P5 涉及数据模型变更，单独 PR

## 验收标准

- [ ] 一个新用户首次记录一天后，看到的统计页**没有任何 +100% 的环比、没有伪造的趋势曲线、没有用单日均摊出来的"每日主导活动"**
- [ ] 本周回顾文案中至少包含一句对比性陈述（含"vs"、"比"、"是...的 N 倍"任一）
- [ ] 任何超预算 ≥50% 的分类，在视觉上明显区别于普通进度条
- [ ] "最长连续记录段"不再返回"睡觉"
- [ ] 所有新增模块在 cold/warming/mature 三种状态下都有明确的呈现规范，且实现一致

## 不要做的事

- 不要改主色板、字体、卡片圆角、留白等已经成型的视觉语言
- 不要引入新的图表库（继续用现有的）
- 不要增加 emoji 或活泼的微交互——CalLens 的气质是"安静"的
- 不要在任何文案里使用感叹号
