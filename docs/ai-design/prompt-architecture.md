# CaILens AI 提示词架构设计备忘

> 2026-05-10 定稿。本文档记录 AI 子系统在产品设计和提示词架构层面的所有已决策事项。

## 一、AI 的角色定位

**"时间镜子的解读者"**——延续"亦是看见自己的镜子"的产品哲学。

- 不是效率教练，不催促用户
- 是一个能看懂用户时间分配的观察者
- 基于用户自己定义的"主要矛盾 / 次要矛盾"分类体系来反馈
- 用"矛盾"这个词本身就暗示用户对自己有判断、有挣扎——AI 必须接住这个调性

## 二、三个核心场景

### 场景 1：周/月回顾（被动触发）

- **输入**：聚合过的时间数据（分类小时数、每天分布、与上周/四周均值对比、连续达成天数、异常值）
- **输出**：三段式结构
  - **看见**（数据陈述，克制）
  - **模式**（一个观察，不做价值判断）
  - **可选的微调**（给一条建议，不多给）

### 场景 2：对话追问（用户主动）

- 关键设计决策：**必须支持 function calling**
- AI 可通过工具反查数据，不依赖初始喂的摘要
- 暴露的查询函数：`query_events`、`get_summary`、`compare_periods`、`find_streaks`

### 场景 3：智能规划（最后做）

- 输出结构化草稿，不直接写入日历
- UI 以半透明/虚线显示建议事件
- 用户单个采纳、批量采纳或拖动调整
- 每个建议事件附简短理由（hover 显示）

## 三、三层提示词架构

### 第一层：系统提示词（产品内置，不可用户修改）

定义 AI 的身份、语气、底线规则。涉及产品调性和安全底线。

### 第二层：用户画像与意图（用户可改，有引导模板）

结构化模板字段：
- 我目前的主要矛盾
- 我目前的次要矛盾
- 我希望保持的状态
- 我已知的坏习惯
- 不希望 AI 提及的话题

新手有引导，老手可切换到高级模式自由编辑。

### 第三层：Skills（用户自定义分析视角）

每个 Skill 是：提示词模板 + 触发条件 + 输出格式。Skill 不是直接发给 AI，而是被系统提示词包装后再发——第一层底线规则永远生效。

## 四、已决策事项

### 4.1 AI 反馈的温度

| 场景 | 温度 |
|---|---|
| 数据陈述 | 完全冷静，不加修饰 |
| 模式识别 | 客观呈现，不做价值判断 |
| 可选建议 | 允许一句温和关怀 |
| 用户情绪追问 | 不附和也不反驳，回到数据 |

系统提示词定调：*"你只反映模式，不评价选择。建议可以温和，但不超过一句话的体谅。"*

### 4.2 Skill 模板形式

**结构化字段 + 自由指令溢出区，两者都有。**

Skill 编辑器字段：
- 名称（文本）
- 描述（文本）
- 触发方式（下拉：手动 / 每周自动 / 条件触发）
- 关注分类（多选）
- 时间范围（下拉）
- 核心指令（自由文本，可引用变量）

编辑区旁边实时预览拼接后的完整提示词。新人选选就能用，老手在核心指令里自由发挥。

### 4.3 用户画像的携带策略

**分级携带，不每次全量。**

| 级别 | 内容 | 使用场景 |
|---|---|---|
| 轻量（~100 tokens） | 当前主要矛盾的一句话概括 | 每次对话都带 |
| 标准（~300 tokens） | + 希望保持的状态 + 已知坏习惯 | 周报生成 + Skill 执行 |
| 完整（~500 tokens） | 全部画像字段 | 用户明确要求整体分析 |

画像只能用户自己改，AI 不自动推测或"学习"用户画像。

### 4.4 其他产品决策

- **AI 用量可见**：每次分析显示消耗 token 数和大约费用
- **AI 历史记录**：可查可删，隐私底线
- **Skill 市场**：v1 不做，先让单用户能写能用
- **AI 不主动弹窗**：所有反馈用户主动触发

## 五、推荐开发顺序

1. DeepSeek API 接入 + Key 管理 + 最简单的"本周分析"按钮
2. 对话功能 + function calling
3. 用户画像填写界面（第二层）
4. Skill 系统 MVP（内置 3-5 个官方 Skill）
5. Skill 编辑器（用户从零写）
6. 智能规划（最后，依赖前面所有能力）
